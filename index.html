    Administrator gate

    Before running intent shaping, unlock the operator session with your private key and passphrase. This keeps execution mode and history behavior under a local admin session instead of a fixed account value in the UI.

    Quick operator tutorial

    1. Set your operator account and unlock admin session using your private key.
    2. Select a surface and input intent, then run shaping.
    3. Copy or export payload logs for audit and evidence.
    4. Use route aliases:
      • /mcp/client
      • /mcp/server
      • /api/openai/v1/chat/completions
      • /openaiapi, /opeaiapi, /oprenaiapi

    Operational rule

    The operator boundary is stored as an internal workspace identity. If the session is locked, non-registered operators or parallel windows are rejected before request shaping.

    A local operator lock keeps a single-window workflow active per account session, so parallel windows in the same browser profile are rejected while shaping output.

    This is not a generative chatbot replacement. It is a protocol cockpit where humans become the actor while the interface provides filtered, structured intent output.

    This is a static identity gateway: endpoint behavior and contracts are visible at a glance, and execution is always operator-mediated.
